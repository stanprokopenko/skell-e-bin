INSANE Parallel Coding with Claude Code + Cursor MCP Server

Today, I dove into an exciting experiment: parallel coding using two clients communicating through an MCP server. The idea was to have Client One (Cursor) and Client Two (Cloud Code) collaborate on a single project simultaneously.

Setting Up the MCP Server

I set up an MCP server running on localhost:8765. Both clients connected to this server via the MCP sync bridge tool. This allowed them to communicate by sending messages and updates about their progress.

Communicating Between Clients

To test the setup, I configured system messages for both clients. Client One and Client Two could send and receive messages, ensuring they were on the same page during development. Although it wasn't fully autonomous yet, the basic communication was functioning.

Building a Parallel Coding Project

I used Gemini 2.5 Pro to create a step-by-step plan for both clients. The project was to build a simple photo upload app that stores images in a local directory called "images". Client One focused on the backend, while Client Two worked on the frontend.

I provided each client with their respective plans and set them to run in full automation. They both started working on their parts, creating backend and frontend directories, and coding simultaneously.

Observations and Results

The clients communicated efficiently, exchanging messages about API contracts and integration testing. In a short time, they completed the project. I ran the backend server on localhost:3000 and opened the frontend to test the app.

I uploaded an image, and it showed a preview and confirmed successful upload. Checking the "images" directory, I found the uploaded file, confirming that the app worked as intended.

Future Improvements

While the initial test was successful, there's room for improvement. I aim to enhance the communication between the clients, making it more autonomous and efficient. Specifically, I'd like them to implement code in stages and synchronize after completing certain tasks.

* * *

This experiment demonstrated that parallel coding with two clients communicating through an MCP server is feasible. I'm excited to continue refining this setup and explore its full potential in future projects.
