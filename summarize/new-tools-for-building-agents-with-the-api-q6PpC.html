Our New Built-in Tools

We're excited to make it easier for developers to build reliable and useful agents, systems that act independently to perform tasks on your behalf. Today, we're launching three new built-in tools -

Web Search Tool

The Web Search Tool lets our models access up-to-date, factual information from the internet. It's the same tool powering ChatGPT's search, using a fine-tuned model excellent at retrieving relevant data and citing it clearly.

File Search Tool

With the File Search Tool, you can upload, chunk, and embed your documents for easy retrieval augmented generation over your data. New features include:

  • Metadata filtering: Add attributes to your files to filter down to the most relevant ones.
  • Direct search endpoint: Directly search your vector stores without model-filtered queries.

Computer Use Tool

The Computer Use Tool (Operator in the API) lets you control computers you're operating, virtual machines or legacy applications without API access. Automate tasks and build applications using this tool, which is powered by the same model as Operator in ChatGPT.

The Responses API

To support these tools, we're introducing the Responses API. Unlike the earlier Chat Completions API, which was text-in and text-out, the Responses API is flexible, supports multiple turns, and integrates tools seamlessly. It's a powerful way to build agents capable of complex, multi-step workflows.

Demonstrating the Responses API

We built a personal stylist assistant to showcase the Responses API. By defining our agent with instructions and adding the File Search and Web Search tools, we could:

  • Summarize user preferences using the File Search Tool.
  • Fetch fresh data from the web with the Web Search Tool.
  • Handle multiple tool calls in one API response.

Using the Computer Use Tool, we automated purchasing a jacket, demonstrating the API's ability to interact with external systems.

Introducing the Agents SDK

Building complex applications often requires multiple agents handling different tasks. To simplify this, we've evolved our experimental Swarm SDK into the production-ready Agents SDK. With it, you can:

  • Orchestrate multiple agents for different parts of your application.
  • Implement handoffs, transferring conversations between agents while maintaining context.
  • Monitor and trace agents with built-in tools.
  • Add guardrails and lifecycle events easily.

The Agents SDK is open source and available via pip install openai-agents. A JavaScript version is coming soon.

Looking Ahead

We'll continue supporting the Chat Completions API with new models and capabilities. Some features will be exclusive to the Responses API, but migrating should be straightforward. We'll add more features to the Responses API to match the Assistance API, provide a migration guide, and plan to sunset the Assistance API in 2026.
