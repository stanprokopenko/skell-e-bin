New tools for building agents with the API

Copy Code

OpenAI has introduced a series of new tools and APIs to make it easier for developers to build reliable and useful agents. Agents are systems that can act independently to perform tasks on behalf of users. This initiative aims to streamline the development process by providing comprehensive resources under a unified platform.

New Built-in Tools

Web Search Tool

The Web Search Tool allows models to access up-to-date and factual information from the internet. It powers ChatGPT's search functionality and is driven by a fine-tuned model (GPT-4 Turbo or GPT-4 Turbo Mini). This model excels at:

  • Analyzing large amounts of data retrieved from the web.
  • Identifying relevant pieces of information.
  • Clearly citing sources in responses.

In the SimpleQA benchmark, GPT-4 Turbo achieved a state-of-the-art score of 90%.

File Search Tool

The File Search Tool enables developers to upload, chunk, and embed documents, facilitating easy retrieval over these documents. Two new features have been added:

  • Metadata Filtering: Attach attributes to files for efficient filtering based on query relevance.
  • Direct Search Endpoint: Directly search vector stores without filtering queries through the model first.

Computer Use Tool

The Computer Use Tool allows control over computers that developers are operating, including virtual machines and legacy applications without API access. It automates tasks by enabling models to:

  • Operate on graphical user interfaces.
  • Perform complex multi-step workflows.

This tool includes the Computer Use Model, the same model used by Operator in ChatGPT. It has received positive feedback and has set state-of-the-art benchmarks on:

  • OSWorld
  • WebArena
  • WebVoyager

Introducing the Responses API

OpenAI has developed the Responses API to provide a flexible and robust way to interact with these tools. Unlike the previous Chat Completions API, the Responses API supports:

  • Multiple turns and tool usage.
  • Multimodal inputs and outputs, including images and audio.
  • Advanced reasoning capabilities.

Features of the Responses API

  • Supports built-in tool use.
  • Allows for complex agentic interactions.
  • Enhances developer experience with straightforward integration.

The Agents SDK

To simplify the building of complex applications, OpenAI has launched the open-source Agents SDK. This SDK is an evolution of the previously experimental Swarm SDK, now made production-ready with added features.

Key Features

  • Agent Orchestration: Manage multiple agents handling different tasks within a single application.
  • Python Function Integration: Define tools as regular Python functions with automatic JSON schema generation.
  • Monitoring and Tracing: Built-in tracing UI for debugging and monitoring agent interactions.
  • Handoffs: Seamlessly switch conversations between agents based on the context and required capabilities.
  • Guardrails and Lifecycle Events: Enhance application robustness and manage agent behaviors effectively.

Installation

The Agents SDK can be installed using:

bash pip install openai-multitool-agents

A JavaScript version is also in development.

Assistance API and Future Plans

OpenAI acknowledges the contributions of the Assistance API and plans to integrate its capabilities into the Responses API. The key points include:

  • Continued Support: The Chat Completions API will remain available, supporting new models and capabilities.
  • Migration Guide: OpenAI will provide guidance for migrating applications from the Assistance API to the Responses API.
  • Sunsetting Assistance API: Plans to Sunset the Assistance API by 2026, with ample time and support for developers to transition.

* * *

OpenAI's new tools and APIs mark significant advancements in empowering developers to build sophisticated agents. By unifying tools like the Web Search Tool, File Search Tool, and Computer Use Tool under the Responses API, and providing the Agents SDK, developers now have a powerful toolkit to create agents that can perform complex tasks. OpenAI is committed to continuing this development, with 2025 set to be the year agents go from answering questions to performing actions in the real world.
